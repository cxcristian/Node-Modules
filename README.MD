# M√≥dulos en Node Js

## ¬øQue son los modulos en Node?

Los modulos de node son unidades de codigo _reutilizable_, los cuales nos permiten organizar de forma eficiente las aplicaciones  
y evitar contaminacion a nivel de scope global.  
Existen tres tipos de modulos:  

+ Nativo: Son m√≥dulos integrados en Node.js que proporcionan funcionalidades  
b√°sicas del sistema operativo y del entorno de ejecuci√≥n.
+ Local:Son archivos JavaScript que creamos en nuestro proyecto para organizar el c√≥digo en diferentes funcionalidades.
+ Terceros: Son paquetes desarrollados por la comunidad que se instalan  
a trav√©s de npm y proporcionan funcionalidades adicionales.
>Nota: Cada archivo que se cree en un proyecto se le puede considerar un modulo independiente medellin.

## CommonJs
Node js usa **CommonJS** como sistema tradicional de modulos, basado en **require()**-**module.exports**, funcionando de forma _sincrona_  
y cache que mantiene estado compartido

## Es modules

Es modules es el estandar hoy en dia (EcmaScript 2015), su sintaxis es **import**/**export**, se diferencia del CommonJs con su carga _asincrona_ y an√°lisis estatico.  
Para usar Es modules se debe configurar el proyecto adecuadamente.

### Mini conclusion
Ambos sistemas tienen diferencias en sintaxis, carga y variables globales, pero Node.js permite cierta interoperabilidad.  
Se recomienda usar ES Modules en proyectos nuevos, manteniendo CommonJS en proyectos existentes.

## Funcionamiento del sistema de m√≥dulos
Cuando Node.js encuentra una declaraci√≥n de importaci√≥n, ejecuta un proceso de resoluci√≥n para localizar el m√≥dulo solicitado.  
Este proceso sigue un orden espec√≠fico de b√∫squeda:  
1 M√≥dulos nativos: Node.js primero verifica si el nombre corresponde a un m√≥dulo nativo.  
2 Rutas relativas y absolutas: Si el nombre comienza con ./, ../ o /, Node.js busca el archivo en la ruta especificada.  
**Ej:**

3 Carpeta node_modules: Para nombres que no son rutas, Node.js busca en la carpeta¬† 
node_modules¬†del directorio actual y en las carpetas padre.

## 6Ô∏è‚É£ **Manejando Eventos del Proceso**

### ‚úÖ **Concepto**

Puedes manejar eventos como:

- `exit`: Cuando el proceso est√° a punto de finalizar.
- `SIGINT`: Cuando se presiona `Ctrl + C`.
- `uncaughtException`: Cuando ocurre un error no controlado.

### ‚ñ∂Ô∏è **Ejemplo**

Crea `events-example.js`:

```js
// Ejemplo de manejo de eventos del proceso en Node.js

// Evento 'exit': se ejecuta justo antes de que el proceso termine
process.on('exit', (code) => {
  console.log(`Proceso finalizado con c√≥digo ${code}`);
});

// Evento 'SIGINT': se ejecuta cuando el usuario presiona Ctrl + C en la terminal
process.on('SIGINT', () => {
  console.log('Capturada la se√±al SIGINT (Ctrl + C)');
  process.exit(0); // Finaliza el proceso manualmente
});

// Evento 'uncaughtException': se ejecuta cuando ocurre un error no controlado en el c√≥digo
process.on('uncaughtException', (err) => {
  console.error('Error no capturado:', err);
  process.exit(1); // Finaliza el proceso con c√≥digo de error
});

// Forzamos un error para demostrar el evento 'uncaughtException'
throw new Error('¬°Esto es un error no capturado!');
```

 Explicaci√≥n:
- process.on('exit', callback): Permite ejecutar c√≥digo justo antes de que el proceso termine.
- process.on('SIGINT', callback): Permite capturar la se√±al Ctrl + C y ejecutar c√≥digo antes de salir.
- process.on('uncaughtException', callback): Permite manejar errores no controlados y evitar que el proceso termine abruptamente sin mensaje.

### ‚ñ∂Ô∏è **Ejecutar**

```bash
node events-example.js
```

### ‚ñ∂Ô∏è **Salida**

```

```

Presionando `Ctrl + C`:


## 1Ô∏è‚É£ **Introducci√≥n al M√≥dulo OS**

El m√≥dulo `os` es un m√≥dulo integrado de Node.js. Proporciona una serie de m√©todos y propiedades para obtener informaci√≥n sobre el sistema operativo en el que se est√° ejecutando la aplicaci√≥n Node.js.

No es necesario instalar ning√∫n paquete adicional; solo tienes que requerirlo en tu proyecto.

```jsx

```

## Resumen del m√≥dulo `os` en Node.js

El m√≥dulo `os` de Node.js permite obtener informaci√≥n y caracter√≠sticas del sistema operativo donde se ejecuta la aplicaci√≥n. Es √∫til para crear scripts de monitoreo, herramientas de administraci√≥n y para adaptar el comportamiento de tu programa seg√∫n el entorno.

### Funcionalidades principales

- **Informaci√≥n del sistema operativo:**
  - `os.platform()`: Plataforma del sistema (win32, linux, darwin, etc).
  - `os.arch()`: Arquitectura de la CPU (x64, arm, etc).
  - `os.type()`: Tipo de sistema operativo.
  - `os.release()`: Versi√≥n del sistema operativo.

- **Informaci√≥n de la CPU:**
  - `os.cpus()`: Detalles de cada n√∫cleo de CPU (modelo, velocidad, etc).
  - `os.cpus().length`: N√∫mero de n√∫cleos.

- **Memoria:**
  - `os.totalmem()`: Memoria RAM total (en bytes).
  - `os.freemem()`: Memoria RAM libre (en bytes).

- **Red:**
  - `os.networkInterfaces()`: Informaci√≥n de las interfaces de red.
  - `os.hostname()`: Nombre del host.

- **Usuario y directorios:**
  - `os.userInfo()`: Informaci√≥n del usuario actual.
  - `os.homedir()`: Ruta del directorio home del usuario.
  - `os.tmpdir()`: Ruta del directorio temporal.

- **Tiempos y carga:**
  - `os.uptime()`: Tiempo de actividad del sistema (en segundos).
  - `os.loadavg()`: Promedio de carga del sistema (solo Unix).

### Ejemplo b√°sico

```js
const os = require('os');
console.log('Sistema:', os.platform(), os.arch());
console.log('CPU:', os.cpus()[0].model);
console.log('Memoria total:', (os.totalmem() / 1024 / 1024).toFixed(2), 'MB');
console.log('Usuario:', os.userInfo().username);
```
 
> **Nota:** El m√≥dulo `os` es solo para informaci√≥n del sistema operativo y hardware, no para manipular archivos o procesos.
